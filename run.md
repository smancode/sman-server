# 复用指南与经验沉淀

## 路径概述

**目标**: 统计当天（自然日00:00-24:00）的代码提交活动，以表格形式展示提交次数、代码变更行数、文件数等量化指标，支持按作者分组或总体汇总。

**适用场景**: 日常代码活动跟踪、团队贡献度统计、项目进度量化分析

## 执行策略

1. **时间范围**: 明确为自然日（00:00-24:00），而非24小时滚动窗口
2. **统计维度**: 
   - 总提交次数
   - 代码变更行数（增加/删除分别统计）
   - 修改的文件数
3. **分组方式**: 
   - 按作者分组统计
   - 总体汇总
4. **输出格式**: Markdown表格，包含两行（按作者分组 + 总体汇总）

## 可复用资源

### Python统计脚本
**路径**: (按需创建临时脚本)

**用途说明**: 
- 使用 `git log` 命令获取当天提交记录
- 通过 `git diff --shortstat` 统计代码变更行数
- 按作者分组汇总并生成表格

**核心命令序列**:
```bash
# 获取当天日期
TODAY=$(date +%Y-%m-%d)

# 获取当天提交列表（按作者）
git log --since="$TODAY 00:00" --until="$TODAY 24:00" --pretty=format:"%an" --no-merges | sort | uniq -c

# 统计代码变更行数
git log --since="$TODAY 00:00" --until="$TODAY 24:00" --pretty=format:"%H" --no-merges | \
  while read commit; do
    git diff --shortstat $commit^..$commit
  done | \
  awk '{insertions+=$1; deletions+=$3} END {print insertions, deletions}'
```

## 注意事项

1. **时区问题**: `git log` 使用本地时区，确保统计环境与预期时区一致
2. **合并提交**: 使用 `--no-merges` 过滤合并提交，避免重复统计
3. **文件计数**: 同一文件被多次修改时，需要去重计数
4. **跨时区协作**: 团队成员在不同时区时，约定统一的统计时区基准

## 最佳实践

1. **定期执行**: 建议每日固定时间（如23:59）执行，确保数据完整性
2. **多仓库支持**: 遍历多个仓库时，使用 `find` 或指定仓库列表
3. **历史对比**: 保存每日统计数据到CSV/JSON，便于趋势分析
4. **异常处理**: 
   - 无提交时输出提示而非报错
   - git仓库不存在时给出明确错误信息
5. **扩展功能**: 可添加分支过滤、文件类型过滤等高级选项
